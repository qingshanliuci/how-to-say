# How to Say — 职场表达教练

> 一套给职场新人的表达原则体系，帮你从"靠感觉说话"变成"有标准地表达"。
>
> 兼容 [OpenClaw](https://openclawlab.com)（小龙虾）、[Claude Code](https://docs.anthropic.com/en/docs/claude-code)、[OpenAI Codex CLI](https://developers.openai.com/codex/cli) 三大 AI 编程助手。

---

## 这是什么

**How to Say** 是一个 AI Agent Skill（技能），安装后你的 AI 助手会变成一个**职场表达教练**。

它基于 Patrick Winston "[How to Speak](https://www.youtube.com/watch?v=Unzc731iCUY)" 的理念，包含 **8 大类 70 条原则**，覆盖职场新人最常见的 6 大场景。

**它不是话术模板**，而是一套可反复调用的说话标准：什么必须做，什么绝不能做，什么场景优先什么原则。

---

## 用起来是什么感觉

安装之后，你只需要用自然语言描述你的场景，AI 就会自动激活教练模式：

### 场景 1：准备明天的汇报

```
你：帮我准备明天的周会汇报，要向领导汇报上周进度

AI：[自动激活 how-to-say]
    场景识别：向上汇报
    抽取原则：#5 先说结论 / #35 别让领导替你总结 / #36 汇报四要素 / #38 带选项

    你的汇报应该按这个结构组织：
    进度 → 问题 → 判断 → 需要的支持

    给我你的汇报草稿，我逐条帮你诊断改写。
```

### 场景 2：直接改写一段话

```
你：帮我改改这段话——"作为刚入职不久的新人，在实际工作中我也发现了
    自身不少需要改进的地方，数据分析能力有待提高……"

AI：[自动激活 how-to-say]

    【原文】作为刚入职不久的新人……数据分析能力有待提高……
    【诊断】违反原则 #50（不要把谦逊演成虚弱）、#38（提问题时带选项）
            三个问题全是"我不行"，没有一个带行动方案。
    【改写】数据分析目前停留在罗列阶段，缺少趋势洞察。
            我计划4月完成 Excel 数据透视表和基础 SQL 自学，
            5月起独立产出月度分析报告。
    【原则】#50 谦逊≠虚弱 / #38 带选项
```

### 场景 3：面试准备

```
你：明天面试，怎么回答"你最大的缺点是什么"

AI：[自动激活 how-to-say]
    场景识别：面试回答
    抽取原则：#29 短板要可控有边界 / #30 不要替自己降级 / #28 四段式

    模板：承认一个真实但可控的短板 → 说明你的认知 → 给出改进动作 → 展示进展
```

**核心体验：你说场景，AI 出教练。不用记原则编号，不用翻手册。**

## 安装方法

三大平台都兼容，选你用的那个：

### OpenClaw（小龙虾）— 推荐

> 特别适合装在飞书上使用，职场场景天然匹配。

**方式一：从 ClawHub 安装（推荐）**

```bash
clawhub install how-to-say
```

**方式二：手动安装**

```bash
# 克隆到工作区 skills 目录
git clone https://github.com/qingshanliuci/how-to-say.git
cp -r how-to-say/skill/ ./skills/how-to-say/

# 或安装到全局
cp -r how-to-say/skill/ ~/.openclaw/skills/how-to-say/
```

安装后重启 OpenClaw，说"帮我准备发言"或"怎么说"即可触发。

### Claude Code

```bash
cp -r skill/ ~/.claude/skills/how-to-say/
```

安装后输入 `/how-to-say` 或说"帮我准备发言"触发。

### OpenAI Codex CLI

```bash
# 项目级安装
cp -r skill/ .agents/skills/how-to-say/

# 或全局安装
cp -r skill/ ~/.codex/skills/how-to-say/
```

安装后输入 `$how-to-say` 或说"帮我准备发言"触发。

| 平台 | 显式触发 | 自然语言触发（自动识别） |
|------|---------|----------------------|

## 来源

这套原则体系来自与 ChatGPT 数字人 "Patrick Winston" 的深度对话，由 [井九](https://github.com/qingshanliuci) 整理并结构化。

[Patrick Winston](https://en.wikipedia.org/wiki/Patrick_Winston) 是 MIT 人工智能实验室前主任，他的 "How to Speak" 演讲是 MIT 历史上最受欢迎的讲座之一。我们将 Winston 的演讲表达理论延伸到了中文职场的日常场景——自我介绍、面试、开会、汇报、提问、回答问题——形成了 **8 大类 70 条原则**。

## 快速参考

### 10 条铁律

1. **先说结论。**
2. **一次只讲一个主结论。**
3. **最多讲 2-3 个支撑点。**
4. **所有发言先想听众关心什么。**
5. **例子必须服务观点。**
6. **少说感受，多说判断。**
7. **少说抽象词，多说具体动作和结果。**
8. **说不清时，先停，再重组，不要硬撑。**
9. **汇报时别只讲过程，要讲价值、问题、建议。**
10. **表达不是展示你想了很多，而是让别人记住最关键的那一点。**

### 12 句稳定器句型

紧张时靠这些句型兜底：

| # | 句型 | 适用场景 |
|---|------|---------|
| 1 | 我的结论是…… | 万能开头 |
| 2 | 这个问题我讲两点。 | 组织结构 |
| 3 | 先说结果，再补充原因。 | 汇报 |
| 4 | 核心问题不在 A，而在 B。 | 分析判断 |
| 5 | 我补充一个风险。 | 开会发言 |
| 6 | 我有一个不同判断。 | 表达异议 |
| 7 | 从执行角度看，我更倾向于…… | 建议选择 |
| 8 | 这个方案可行，但前提是…… | 有条件同意 |
| 9 | 如果今天要推进，我建议先做…… | 推动决策 |
| 10 | 我收一下，我真正想表达的是…… | 自救拉回 |
| 11 | 这件事目前最大的卡点是…… | 暴露问题 |
| 12 | 我现在没有完整答案，但初步判断是…… | 坦诚但不塌陷 |

### 6 大场景速查

| 场景 | 核心目标 | 结构模板 |
|------|---------|---------|
| **自我介绍** | 让人记住你 | 我是谁 → 我擅长什么 → 我能贡献什么 |
| **面试回答** | 证明你能干 | 结论 → 过程 → 结果 → 反思 |
| **开会发言** | 推动会议前进 | "我补充一个X" → 一句结论 → 一个支撑 |
| **向上汇报** | 让领导快速判断 | 进度 → 问题 → 判断 → 需要的支持 |
| **提问** | 体现思考深度 | 我理解的现状 → 我的疑问 → 想确认的问题 |
| **回答问题** | 准确命中问题类型 | 先给结论 → 分层展开 → 收束落地 |
