# How to Say — 职场表达教练

> 一套给职场新人的表达原则体系，帮你从"靠感觉说话"变成"有标准地表达"。

## 这是什么

**How to Say** 是一个基于 Patrick Winston "[How to Speak](https://www.youtube.com/watch?v=Unzc731iCUY)" 理念的职场表达训练框架，专为中文职场环境设计。

它不是话术模板，而是一套**可反复调用的说话原则**：什么必须做，什么绝不能做，什么场景优先什么原则。

核心理念：

- **职场表达的目标不是"把话说出来"，而是让对方更快理解、更容易决策、更愿意信任你**
- 思考过程可以复杂，表达结果必须压缩
- 没有一套固定标准的人，临场一定不稳定

## 来源

这套原则体系来自与 ChatGPT 数字人 "Patrick Winston" 的深度对话，由 [井九](https://github.com/qingshanliuci) 整理并结构化。Patrick Winston 是 MIT 人工智能实验室前主任，他的 "How to Speak" 演讲是 MIT 历史上最受欢迎的讲座之一。

我们将 Winston 的演讲表达理论延伸到了中文职场的日常场景——自我介绍、面试、开会、汇报、提问、回答问题——形成了 **8 大类 70 条原则**。

## 快速上手

### 10 条铁律

如果只记 10 条，记这些：

1. **先说结论。**
2. **一次只讲一个主结论。**
3. **最多讲 2-3 个支撑点。**
4. **所有发言先想听众关心什么。**
5. **例子必须服务观点。**
6. **少说感受，多说判断。**
7. **少说抽象词，多说具体动作和结果。**
8. **说不清时，先停，再重组，不要硬撑。**
9. **汇报时别只讲过程，要讲价值、问题、建议。**
10. **表达不是展示你想了很多，而是让别人记住最关键的那一点。**

### 12 句稳定器句型

紧张时靠这些句型兜底，不靠灵感：

1. 我的结论是……
2. 这个问题我讲两点。
3. 先说结果，再补充原因。
4. 核心问题不在 A，而在 B。
5. 我补充一个风险。
6. 我有一个不同判断。
7. 从执行角度看，我更倾向于……
8. 这个方案可行，但前提是……
9. 如果今天要推进，我建议先做……
10. 我收一下，我真正想表达的是……
11. 这件事目前最大的卡点是……
12. 我现在没有完整答案，但初步判断是……

### 6 大场景速查

| 场景 | 核心目标 | 结构模板 |
|------|---------|---------|
| **自我介绍** | 让人记住你 | 我是谁 → 我擅长什么 → 我能贡献什么 |
| **面试回答** | 证明你能干 | 结论 → 过程 → 结果 → 反思 |
| **开会发言** | 推动会议前进 | "我补充一个X" → 一句结论 → 一个支撑 |
| **向上汇报** | 让领导快速判断 | 进度 → 问题 → 判断 → 需要的支持 |
| **提问** | 体现思考深度 | 我理解的现状 → 我的疑问 → 想确认的问题 |
| **回答问题** | 准确命中问题类型 | 先给结论 → 分层展开 → 收束落地 |

## 作为 Claude Code Skill 使用

如果你使用 [Claude Code](https://docs.anthropic.com/en/docs/claude-code)，可以直接将 `skill/` 目录安装为 Skill：

```bash
# 复制到 Claude Code skills 目录
cp -r skill/ ~/.claude/skills/how-to-say/
```

安装后，在 Claude Code 中输入 `/how-to-say` 或说"帮我准备发言"即可触发教练模式：

- 识别你的场景（汇报？面试？开会？）
- 抽取最相关的原则
- 给出结构模板
- 对你的表达逐条诊断改写
- 练习循环直到通过
